@immutable
abstract class Widget extends DiagnosticableTree {
	···
	/// Whether the `newWidget` can be used to update an [Element] that currently
  /// has the `oldWidget` as its configuration.
  ///
  /// An element that uses a given widget as its configuration can be updated to
  /// use another widget as its configuration if, and only if, the two widgets
  /// have [runtimeType] and [key] properties that are [operator==].
  ///
  /// If the widgets have no key (their key is null), then they are considered a
  /// match if they have the same type, even if their children are completely
  /// different.
  static bool canUpdate(Widget oldWidget, Widget newWidget) {
    return oldWidget.runtimeType == newWidget.runtimeType
        && oldWidget.key == newWidget.key;
  }

	···
}

根据注释翻译：

  • canUpdate() 是判断是否可以使用‘newWidget’ 来更新当前以’oldWidget’作为配置的Element
    /// 简而言之：就是Element是否能够重用，通过使用newWidget配置文件刷新的方式

  • 只有当两个widget的runtimeType和key值相同的时候，Element 可以通过使用另外一个widget配置文件更新再次被使用

    /// 简而言之：如果Element要重用，那么widget的runtimeType和key的值必须相同

  • widget没有key（默认是null）,如果它们具有相同的runtimeType，则认为它们是匹配的，即使他们的子widget完全不同

案例： 点击按钮，交互两个色块的位置

import 'package:flutter/material.dart';
import 'dart:math';

class RWKeyDemoPage extends StatefulWidget {
  @override
  _RWKeyDemoPageState createState() => _RWKeyDemoPageState();
}

class _RWKeyDemoPageState extends State<RWKeyDemoPage> {
  List<Widget> widgets = [
    StatelessContainer(),
    StatelessContainer(),
  ];

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('KeyDemo'),
      ),
      body: Center(
        child: Row(
          mainAxisAlignment: MainAxisAlignment.center,
          children: widgets,
        ),
      ),
      floatingActionButton: FloatingActionButton(onPressed: () {
        widgets.insert(0, widgets.removeAt(1));
        setState(() {});
      }),
    );
  }
}

/// StatelessWidget 色块
class StatelessContainer extends StatelessWidget {
  final Color color = RandomColor().randomColor();
  
  @override
  Widget build(BuildContext context) {
    return Container(
      width: 100,
      height: 100,
      color: color,
    );
  }
}

色块使用StatelessWidget来定义，点击按钮是能够现实需求，完成两个色快位置交换。

但是如果将色块替换成StatefulWidget定义，却发现点击按钮后没有效果，并且色块会rebuild。

为什么点击了色块颜色不会交换？

根据源码的canUpdate定义，两个StatefulContainer都没有定义key并且它们的runtimeType是一样的，那么其Element是能复用的，并且使用新的widget配置文件刷新就行，而StatefulContainer中color是保存在state中，
