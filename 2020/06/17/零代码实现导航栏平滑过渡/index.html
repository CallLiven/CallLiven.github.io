前言

以上是当前项目中视频文章详情页导航栏在切换的时候，出现过渡不友好的情况，相对于今天头条，就是渣渣app。同时文章详情页的样式也是有多种的，有一种是视频置顶导航栏透明的，有一种的是导航栏显示不透明，为了简化文章详情的入口，所以现在所有的文章详情样式都是在同一个ViewController中显示的，也就是导航栏需要动态的修改隐藏或者显示的样式。

正文

项目已迭代多个版本，所以如果是重构导航栏，那工作比较大，不符合实际的情况，所以选择在原有的基础上做修改。

视频文章详情样式的需求：视频是置于顶部固定不变，不会顺着上下滑动

原代码的做法就比较粗暴，直接将导航栏隐藏，然后自定义返回按钮；由于隐藏了导航栏，那么系统自带的侧滑返回的手势就关闭了，所以引入了FDFullscreenPopGesture的第三方库。

修改后的代码如下：

#import "SecondViewController.h"
#import <FDFullscreenPopGesture/UINavigationController+FDFullscreenPopGesture.h>
#import "ThirdViewController.h"

@implementation SecondViewController

- (void)viewDidLoad {
    [super viewDidLoad];
    
    self.title = @"第二个界面";
    self.view.backgroundColor = [UIColor yellowColor];
    
    UIButton *actionBtn = [UIButton buttonWithType:UIButtonTypeCustom];
    actionBtn.frame = CGRectMake(20, 200, [UIScreen mainScreen].bounds.size.width-40, 60);
    actionBtn.backgroundColor = [UIColor blueColor];
    [actionBtn setTitle:@"动态修改导航栏的隐藏或显示" forState:UIControlStateNormal];
    [actionBtn addTarget:self action:@selector(btnAction) forControlEvents:UIControlEventTouchUpInside];
    [self.view addSubview:actionBtn];
    
    
    UIButton *jumpBtn = [UIButton buttonWithType:UIButtonTypeCustom];
    jumpBtn.frame = CGRectMake(20, 300, [UIScreen mainScreen].bounds.size.width-40, 60);
    jumpBtn.backgroundColor = [UIColor blueColor];
    [jumpBtn setTitle:@"跳转" forState:UIControlStateNormal];
    [jumpBtn addTarget:self action:@selector(jumpAction) forControlEvents:UIControlEventTouchUpInside];
    [self.view addSubview:jumpBtn];
}


/// 这是关键代码部分
- (void)btnAction {
    BOOL isHidden = self.navigationController.navigationBarHidden;
    /// 这个方法是监听viewWillAppear调用时候设置setNavigationBarHidden的值
    self.fd_prefersNavigationBarHidden = !isHidden;
    /// 这是即使设置导航栏的隐藏
    [self.navigationController setNavigationBarHidden:!isHidden];
    /// 同时设置这两个属性是为了在侧滑的时候，如果只是滑动一小段，但又没跳转，导致导航栏的隐藏显示不统一或跳闪的问题
}


- (void)jumpAction {
    [self.navigationController pushViewController:[ThirdViewController new] animated:YES];
}


@end
