二、基本原理

AVPlayer 没有提供API获取播放器下载的数据，所以无法直接获取缓存，但是可以通过设置AVAssetResourceLoader代理的方式，自己使用NSURLSession下载数据，再传给播放器播放，同时缓存在本地，下次播放的时候可以使用。

AVAssetResourceLoader 只有在AVURLAsset加载不能识别的URL资源的时候，才会回调代理方法，所以在初始化AVURLAsset的时候把目标视频URL地址的scheme替换成系统不能识别的schemen。

（1）AVAssetResourceLoaderDelegate 代理有两个必须要实现的方法

要求加载资源的代理方法

返回YES，表示该代理类现在可以处理该请求，需要在这里保存loadingReques并开启下载数据的任务

代理方法中AVAssetResourceLoadingRequest类非常关键，以下是该类常用的属性和方法

@property (nonatomic, readonly) NSURLRequest *request;
@property (nonatomic, readonly, nullable) AVAssetResourceLoadingContentInformationRequest *contentInformationRequest;
@property (nonatomic, readonly, nullable) AVAssetResourceLoadingDataRequest *dataRequest;

- (void)finishLoading;
- (void)finishLoadingWithError:(nullable NSError *)error;

request 代表原始的请求，获取资源请求地址

contentInformationRequest 播放资源相关的信息，比如资源格式contentType、资源长度大小contentLength、是否支持分片下载byteRangeAccessSupported等

dataRequest 表示请求资源数据的范围。由于AVPlayer会触发分片下载的策略，也就是说一个视频资源可能对应多个AVAssetResourceLoadingRequest，可以从这个属性获取每个AVAssetResourceLoadingRequest请求数据的范围

- (void)finishLoading 如果AVAssetResourceLoadingRequest要求下载的数据都下载完毕则调用此方法完成下载

- (void)finishLoadingWithError:(nullable NSError *)error; 如果AVAssetResourceLoadingRequest对应的下载发生失败，则调用此方法

（2）流程原理

AVPlayer是分片式下载策略，可能会发起多个LoadingRequest请求，但触发下载的时候，会先发起一个Range为0~2的数据请求，获取视频资源的描述信息，比如文件类型、文件大小，这需要我们填充到AVAssetResourceLoadingRequest类中的contentInformationRequest属性，而获取到的data填充至AVAssetResourceLoadingRequest的dataRequest，dataRequest属性中有个方法可以接收增量连续的data数据:- (void)respondWithData:(NSData *)data，当AVAssetResourceLoadingRequest要求下载的数据都下载完毕，则调用- (void)finishLoading完成请求，并且AVPlayer会在调用finishLoading时判断接下来要怎么处理，比如视频文件类型是否支持、是否要发起下一个LoadingRequest等

（3）重试机制

AVAssetResourceLoader在执行加载的时候，会时不时触发取消下载方法\- (void)resourceLoader:(AVAssetResourceLoader *)resourceLoader didCancelLoa，然后重新发起加载请求策略，如果下载了部分，重新发起的下载请求会从还没下载的部分开始。

#pragma mark - AVAssetResourceLoaderDelegate
/// 要求加载资源的代理方法
/// 返回YES，表示该代理类现在可以处理该请求，需要在这里保存loadingReques并开启下载数据的任务
/// 下载回调中拿到响应数据后再对loadingRequest进行填充
/// 如果返回NO，则表示当前代理不能处理，视频数据需要AVPlayer自己处理(但是之前视频URL的scheme被设置自定义的，所以AVPlayer不能识别，最后导致 AVPlayerItemStatusFailed)
- (BOOL)resourceLoader:(AVAssetResourceLoader *)resourceLoader shouldWaitForLoadingOfRequestedResource:(AVAssetResourceLoadingRequest *)loadingRequest {
    NSString *requestURL = loadingRequest.request.URL.absoluteString;
    if ([requestURL containsString:kScheme] && ![_requestsArray containsObject:loadingRequest]) {
        [_requestsArray addObject:loadingRequest];
        
        if (self.task == nil) {
            NSString *orginURL = [requestURL stringByReplacingOccurrencesOfString:kScheme withString:@""];
            NSMutableURLRequest *request = [[NSMutableURLRequest alloc]initWithURL:[NSURL URLWithString:orginURL] cachePolicy:(NSURLRequestUseProtocolCachePolicy) timeoutInterval:20];
            self.task = [self.session dataTaskWithRequest:request];
            [self.task resume];
        }
        
        return YES;
    }

    return NO;
}


/// AVAssetResourceLoader取消了本次请求
/// 把loadingRequest移除下载任务回调列表(停止填充数据)
- (void)resourceLoader:(AVAssetResourceLoader *)resourceLoader didCancelLoadingRequest:(AVAssetResourceLoadingRequest *)loadingRequest {
    [_requestsArray removeObject:loadingRequest];
}


#pragma mark - NSURLSessionDelegate
- (void)URLSession:(NSURLSession *)session dataTask:(NSURLSessionDataTask *)dataTask didReceiveResponse:(NSURLResponse *)response completionHandler:(void (^)(NSURLSessionResponseDisposition))completionHandler {
    completionHandler(NSURLSessionResponseAllow);
    if ([response isKindOfClass:NSHTTPURLResponse.class]) {
        NSHTTPURLResponse *httpResponse = (NSHTTPURLResponse *)response;
        NSString *acceptRange = httpResponse.allHeaderFields[@"Accept-Ranges"];
        self.byteRangeAccessSupported = [acceptRange isEqualToString:@"bytes"];
    }
    
    NSString *mimeType = [response MIMEType];
    CFStringRef contentType = UTTypeCreatePreferredIdentifierForTag(kUTTagClassMIMEType, (__bridge CFStringRef)(mimeType), NULL);

    self.contentLength = response.expectedContentLength;
    self.contentType = CFBridgingRelease(contentType);
    
    NSLog(@"视频大小：%f M",response.expectedContentLength/1024.0f/1024.0f);
}


- (void)URLSession:(NSURLSession *)session dataTask:(NSURLSessionDataTask *)dataTask didReceiveData:(NSData *)data {
    // 缓存下载的data
    [self.downloadData appendData:data];
    // 拼接下载的data
    [self processRequestData];
    
    NSLog(@"已下载：%f ，正在下载：%f",self.downloadData.length/1024.0,data.length/1024.0);
}


- (void)URLSession:(NSURLSession *)session task:(NSURLSessionTask *)task didCompleteWithError:(NSError *)error {
    NSLog(@"下载完成");
}



#pragma mark - ProcessRequestData
- (void)processRequestData {
    NSMutableArray *requestCompleted = [NSMutableArray array];
    for (AVAssetResourceLoadingRequest *loadingRequest in self.requestsArray) {
        loadingRequest.contentInformationRequest.contentLength = self.contentLength;
        loadingRequest.contentInformationRequest.contentType = self.contentType;
        loadingRequest.contentInformationRequest.byteRangeAccessSupported = self.byteRangeAccessSupported;
        
        BOOL didRespondCompletely = [self respondWithDataForRequest:loadingRequest.dataRequest];
        if (didRespondCompletely) {
            [requestCompleted addObject:loadingRequest];
            [loadingRequest finishLoading];
        }
    }
    [self.requestsArray removeObjectsInArray:requestCompleted];
}



/// 判断loadingRequest要求下载的数据是否全部下载完毕
/// 以及将下载的数据data拼接到dataRequest当中
/// @param dataRequest dataRequest
- (BOOL)respondWithDataForRequest:(AVAssetResourceLoadingDataRequest *)dataRequest {
    long long startOffset = dataRequest.requestedOffset;
    if (dataRequest.currentOffset != 0) {
        startOffset = dataRequest.currentOffset;
    }
    
    if (self.downloadData.length < startOffset) {
        return NO;
    }
    
    NSUInteger unreadBytes = self.downloadData.length - startOffset;
    NSUInteger numberOfBytesToRespondWith = MIN(dataRequest.requestedLength, unreadBytes);
    
    [dataRequest respondWithData:[self.downloadData subdataWithRange:NSMakeRange(startOffset, numberOfBytesToRespondWith)]];
    
    long long endOffset = startOffset + dataRequest.requestedLength;
    BOOL didResponsdComplete = self.downloadData.length >= endOffset;
    
    return didResponsdComplete;
}



#pragma mark - KVO
- (void)observeValueForKeyPath:(NSString *)keyPath ofObject:(id)object change:(NSDictionary<NSKeyValueChangeKey,id> *)change context:(void *)context {
    if ([keyPath isEqualToString:@"status"]) {
        switch (self.playerItem.status) {
            case AVPlayerItemStatusUnknown: {
                // 未知状态，不能播放
                NSLog(@"AVPlayerItemStatusUnknown");
            }
                break;
            case AVPlayerItemStatusReadyToPlay: {
                // 准备完毕，可以播放
                // 此方法可以在视频未播放的时候，获取视频的总时长(备注：一定要在AVPlayer预加载状态status是AVPlayerItemStatusReadyToPlay才能获取)
                // NSLog(@"total %f",CMTimeGetSeconds(self.playerItem.asset.duration));
                [self.player play];
                NSLog(@"AVPlayerItemStatusReadyToPlay");
            }
                break;
            case AVPlayerItemStatusFailed: {
                // 加载失败，网络或者服务器出现问题
                NSLog(@"AVPlayerItemStatusFailed");
            }
                break;
            default:
                break;
        }
    }
}

    if (!_session) {
        NSURLSessionConfiguration *configuration = [NSURLSessionConfiguration defaultSessionConfiguration];
        _session = [NSURLSession sessionWithConfiguration:configuration delegate:self delegateQueue:[NSOperationQueue mainQueue]];
    }

}

突然心情开朗，终于知道第二种方式问题的关键点(测试代码下载地址：链接:https://pan.baidu.com/s/16hFRJaDgFJzj2IQadaW87w 密码:z0cw）

前面说的resourceLoader触发下载，会先发起一个range为0~2的request，来获取视频资源的详情，比如文件长度，接着就是将获取的文件详情填充至contentInformationRequest，到这为止是没有错的，但是，contentInformationRequest是相对于整个视频资源文件的，这非常关键，也就是说设置的文件格式、文件长度都是指整个文件，而并不是指每一个发起的AVAssetResourceLoadingRequest的长度。

所以在第二种方式中，由于每一个AVAssetResourceLoadingRequest单独发起一个request请求，并且设置request请求数据范围range值，所以获取到的文件长度是对应于当前请求范围range，并不是指整个文件的长度。因此才会在发起第一个AVAssetResourceLoadingRequest请求range为0~2后，就没有继续发起第二个AVAssetResourceLoadingRequest了，因为resourceLoader会判断整个视频资源已经下载完成了。

激动~终于知道原因了，可以继续往前走，太不容易了，每一个小细节都可能导致失败

好了，问题来了，那要怎么分片下载，并且获取整个视频文件的长度？

貌似很矛盾的一个问题，其实是文件长度的取的字段有问题，通过答应服务器响应的NSURLResponse如下图，使用，代码改为如下就能解决问题了

- (void)URLSession:(NSURLSession *)session dataTask:(NSURLSessionDataTask *)dataTask didReceiveResponse:(NSURLResponse *)response completionHandler:(void (^)(NSURLSessionResponseDisposition))completionHandler {
    completionHandler(NSURLSessionResponseAllow);
    BOOL byteRangeAccessSupported = NO;
    long long contentLength = 0;
    if ([response isKindOfClass:NSHTTPURLResponse.class]) {
        NSHTTPURLResponse *httpResponse = (NSHTTPURLResponse *)response;
        NSString *acceptRange = httpResponse.allHeaderFields[@"Accept-Ranges"];
        byteRangeAccessSupported = [acceptRange isEqualToString:@"bytes"];
        contentLength = [[[httpResponse.allHeaderFields[@"Content-Range"] componentsSeparatedByString:@"/"] lastObject] longLongValue];
    }
    
    NSString *mimeType = [response MIMEType];
    CFStringRef contentType = UTTypeCreatePreferredIdentifierForTag(kUTTagClassMIMEType, (__bridge CFStringRef)(mimeType), NULL);
    if (contentType && contentLength>0) {
        self.loadingRequest.contentInformationRequest.contentLength = contentLength;
        self.loadingRequest.contentInformationRequest.contentType = CFBridgingRelease(contentType);
        self.loadingRequest.contentInformationRequest.byteRangeAccessSupported = byteRangeAccessSupported;
    }
    NSLog(@"视频大小：%f M",response.expectedContentLength/1024.0f/1024.0f);
}
